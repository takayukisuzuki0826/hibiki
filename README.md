# Hibiki（響）— 周波数で、ととのう。

ANIMA（Anima: Binaural Beats）の仕組みを **完璧に再現したオリジナルのWebアプリ**。
バイノーラルビート／アイソクロニックトーン／ソルフェジオ周波数で脳波をやさしく誘導し、
**睡眠・リラックス・集中・瞑想・不安解放・耳鳴り緩和**をサポートします。
スマホのホーム画面に追加できるPWA。音はすべて**ブラウザ内でリアルタイム合成**（音源ファイル不要・著作権クリーン）。

---

## 仕組み（ANIMAと同じ原理）

| 方式 | 原理 | ヘッドホン |
|------|------|-----------|
| **バイノーラルビート** | 左右の耳でわずかに違う周波数を鳴らすと、脳がその差を「うなり」として知覚し、脳波がその周波数に同調しやすくなる（周波数追従反応 FFR） | **必須** |
| **アイソクロニックトーン** | 単一の音を一定間隔でON/OFFして脳波を誘導 | 不要 |
| **ソルフェジオ周波数** | 396/432/528/639Hz など、心地よいとされる純音 | 不要 |

### 脳波帯域
- **デルタ波 0.5–4Hz** … 深い睡眠
- **シータ波 4–8Hz** … 瞑想・入眠
- **アルファ波 8–14Hz** … リラックス集中
- **ベータ波 14–30Hz** … 覚醒・集中
- **ガンマ波 30–100Hz** … 高度な認知

環境音（雨・波・森・夜・桜）とノイズ（ホワイト/ピンク/ブラウン）も**合成**で重ねられます。

---

## 機能

- 7カテゴリ・18プリセット（睡眠／リラックス／集中／瞑想／不安解放／耳鳴り緩和／内なるバランス）
- **ミキサー**：ビート／ノイズ／環境音／マスターの音量を個別調整
- **スリープタイマー**：5/10/15/30/45/60分・∞、終了時はフェードアウト
- **お気に入り保存**（ANIMAにない改善点・ブラウザに永続化）
- **カスタム周波数モード**：キャリア20–500Hz・ビート0.5–40Hzを自分で設定
- **科学的根拠の常時表示**（誇大表現なし・薬機法/景表法に配慮）
- **PWA**：ホーム画面追加・オフライン起動・画面オフ防止（Wake Lock）
- スマホ最適化（safe-area対応・タップ44px・瞑想的ダークUI）

## 使い方

### ローカルで試す（PC）
```bash
cd hibiki
python3 -m http.server 8137
# ブラウザで http://localhost:8137/ を開く
```
> ⚠️ `file://` で直接開くとESモジュールが読めません。必ずHTTPサーバー経由で。
> バイノーラルビートは**ヘッドホン推奨**（左右の周波数差で効くため）。

### スマホで使う
1. 下記いずれかでホスティング（HTTPS必須・PWAのため）
2. スマホのブラウザで開く → 「ホーム画面に追加」
3. アプリのように全画面で起動。オフラインでも動作。

#### デプロイ（推奨：GitHub Pages 無料）
```bash
# GitHubにpush後、Settings > Pages で main / root を公開
# → https://<user>.github.io/hibiki/ でアクセス
```
他に Cloudflare Pages / Netlify / Vercel いずれも静的サイトとしてそのまま動きます（ビルド不要）。

## 技術構成（ビルドレス・依存ゼロ）

```
hibiki/
├── index.html              UI（DOM契約に準拠）
├── css/styles.css          瞑想的ダークUI（グラスモーフィズム・オーロラ背景）
├── js/
│   ├── audio-engine.js     Web Audio APIエンジン（全音合成・心臓部）
│   ├── presets.js          プリセット/帯域/科学テキスト
│   ├── visualizer.js       Canvasビジュアライザー（呼吸する波紋）
│   └── app.js              UI配線・状態管理・localStorage
├── manifest.webmanifest    PWAマニフェスト
├── sw.js                   Service Worker（オフライン）
├── icons/                  アイコン（依存なしで自前生成）
└── DESIGN.md               設計契約書（Interface-First Protocol）
```

- 外部ライブラリ・CDN依存ゼロ。Vanilla JS（ESM）+ Web Audio API + Canvas。
- 音源ファイルを一切持たず、全周波数・ノイズ・環境音を**実行時に合成**。

## 開発（シールド／Avengers体制で構築）

設計契約 `DESIGN.md` を不変Protocolとして固定し、6ワーカーがファイル分担で並列実装
（Interface-First）。多重Critic（コード／設計／視覚）でレビュー → 修正 → 実機検証の
パイプラインで品質を担保。
